How the signal is built

Each coin's signal is a label, not a forecast. It is assembled only from data you can check:

  • Momentum — whether the 24h, 7d and 30d changes point the same way.
  • Volatility — how much the real 7-day price series has swung (coefficient of variation).
  • Turnover — 24h volume relative to market cap (how much is actually trading).

We now publish two forward-looking numbers in each coin's detail — a 24h up-probability and an 80% range — but only because each is logged the moment it's made and automatically scored 24 hours later against a random-walk baseline. You can audit whether we beat a coin flip; when we don't, the scoreboard says so plainly. We still refuse any number that can't be checked: never a precise "expected % change" (provably no better than assuming no change), and derivatives/on-chain metrics (open interest, funding, liquidations, flows) stay marked not available until a real source is connected — never invented.

Crypto Pulse is descriptive analytics of public market data. It does not predict prices, does not give financial or investment advice, and makes no profit claims. Data from CoinGecko's public API; figures move continuously. Think for yourself.
